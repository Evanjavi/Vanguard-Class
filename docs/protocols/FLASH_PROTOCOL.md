### D. Verification Efficiency (Terminal Sovereignty)

**Trigger:** Verification of user flows.
**Problem:** Browser Tool usage causes "Confirmation Overflow" (manual user clicks).
**Rule:** **OMNIBUS BAN** on Browser Tool for verification.

**Action:**

1.  **Write:** Create a Node.js script (e.g., `scripts/verify_[feature].js`) using `fetch` or a headless browser library.
2.  **Run:** Execute via Terminal (`node scripts/verify_...`).
3.  **Result:** Automated Pass/Fail. Zero manual clicks after script launch.

> **Directive:** If you cannot verify it via terminal, **WRITE A TOOL** that can. Do not ask the user to verify manually.
