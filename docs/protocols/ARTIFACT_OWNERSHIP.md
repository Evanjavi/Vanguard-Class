# Artifact Ownership Protocol

**Version:** 1.0  
**Last Updated:** 2026-01-10  
**Author:** Opus (XO)

---

## Purpose

Defines which voice owns which documentation artifacts to prevent duplication, drift, and cross-conversation conflicts.

---

## Ownership Matrix

| Artifact Type             | Owner | Location                                        |
| ------------------------- | ----- | ----------------------------------------------- |
| **THE_BOOK.md**           | Opus  | `[Project]/THE_BOOK.md`                         |
| **Project Walkthroughs**  | Opus  | `[Project]/docs/walkthroughs/`                  |
| **Implementation Plans**  | Opus  | `[Project]/.antigravity/implementation_plan.md` |
| **Task Trackers**         | Opus  | `[Project]/.antigravity/task.md`                |
| **Chorus Reviews**        | Opus  | `[Project]/.antigravity/reviews/`               |
| **Time Capsules**         | Atlas | `[Project]/Chorus Documents/capsules/`          |
| **Playbook Walkthroughs** | Atlas | `VanguardPlaybook/docs/walkthroughs/`           |
| **Playbook Protocols**    | Atlas | `VanguardPlaybook/docs/protocols/`              |
| **Security Audits**       | Vex   | `[Project]/Chorus Documents/audits/`            |

---

## Rationale

### Opus (XO, Strategic Planner)

- **Writes narrative and strategy:** THE_BOOK, implementation plans, project walkthroughs
- **Coordinates execution:** Task trackers, Chorus Reviews
- **Voice:** Prosaic, strategic, forward-looking

### Atlas (Memory Officer)

- **Guards institutional memory:** Playbook protocols, playbook walkthroughs
- **Prevents drift:** Time Capsules, cross-project documentation
- **Voice:** Archival, precise, pattern-focused

### Vex (Security Officer)

- **Owns security artifacts:** Audit reports, threat models
- **Direct line to Commander:** Security escalations bypass normal flow

---

## Conflict Resolution

If ownership is unclear:

1. **Project-specific narrative** → Opus
2. **Cross-project/meta documentation** → Atlas
3. **Security-related** → Vex
